Description

Game handling, primarily through Transforms.

If a status Transform ends in an apostrophe, it uses a custom display name. If a trap Transform ends in an apostrophe, it is hidden from both players.

A common pattern in higher-order Transforms is (Int -> Transform) -> Int. This modifies the amount passed to a Transform that requires an amount, such as damage (how much?) or applyDur (how long?). (Int -> Transform) is the function, and the following Int is the base amount.

For example, withI "Example Status" 25 damage 50 results in "damage 50, and an additional 25 if the user has Example Status." This way, withI "Example Status" 25 looks like an adverb applied to damage 50, even though damage and 50 are actually two separate arguments of withI.

Turn handling

act :: [Affected] -> Game -> (Act, Either (Slot, Slot) StdGen) -> Game Source #

Performs a single Act. If Right, this is a new action, in which case random targets are chosen by StdGen. If Left, this is a continuation of some previous action, in which case its random targets carry over from the previous action.

runTurn :: Player -> [Act] -> StdGen -> Game -> Game Source #

The game engine's main function. Performs Acts and Channels; applies effects from Bombs, Barriers, Delays, and Traps; decrements all TurnBased data; adds ChannelTags; and resolves Chakras for the next turn.

updateChakra :: Player -> [Int] -> Game -> Game Source #

Uses random integers to give a Player chakras at the start of their turn.
